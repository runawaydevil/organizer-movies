# Contributing

Contributions are welcome. Author: Pablo Murad (runawaydevil). 2025-2026.

## How to contribute

1. Fork the repo, create a branch.
2. Make changes; keep style consistent (e.g. Black if used).
3. Run tests if present: `pytest` or `pip install -r requirements-dev.txt` then test.
4. Open a pull request with a short description of the change.

## Scope

- Bug fixes, docs, tests, and small features are ideal.
- For larger features, open an issue first to align with the project direction.
